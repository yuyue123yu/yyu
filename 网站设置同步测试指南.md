# 网站设置同步测试指南

## 🎯 测试目标
验证在租户管理系统修改网站设置后，前端公共网站能够实时同步显示。

## 📋 测试前准备

### 1. 确认服务器运行
- 开发服务器地址：`http://localhost:3000`
- 状态：✅ 正常运行

### 2. 打开浏览器开发者工具
- 按 `F12` 或右键 → 检查
- 切换到 `Console` 标签
- 清空控制台日志

## 🧪 测试步骤

### 步骤 1：查看当前设置

1. **访问公共网站首页**
   ```
   http://localhost:3000
   ```

2. **记录当前显示的信息**
   - Header中的网站名称：`_____________`
   - Footer中的网站名称：`_____________`
   - Footer中的网站描述：`_____________`
   - Footer中的联系邮箱：`_____________`
   - Footer中的联系电话：`_____________`

3. **检查控制台日志**
   - 应该看到：`✅ 网站设置已加载: {siteName: "...", ...}`

### 步骤 2：修改设置

1. **访问租户管理系统设置页面**
   ```
   http://localhost:3000/admin/settings
   ```

2. **修改网站设置**
   - 网站名称：改为 `马来西亚法律网` 或其他名称
   - 网站描述：改为 `专业法律咨询平台` 或其他描述
   - 联系邮箱：改为新的邮箱地址
   - 联系电话：改为新的电话号码

3. **保存设置**
   - 点击页面顶部的 "保存设置" 按钮
   - 等待成功提示：`设置保存成功！页面将自动刷新以应用更改。`
   - 页面会自动刷新

4. **检查Admin页面**
   - 刷新后，检查左侧边栏顶部
   - 网站名称应该已更新为新名称 + " Admin"

### 步骤 3：验证公共网站同步

1. **刷新公共网站首页**
   ```
   http://localhost:3000
   ```
   - 使用硬刷新：`Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac)

2. **验证Header更新**
   - 检查页面顶部Logo旁边的网站名称
   - ✅ 应该显示新的网站名称

3. **验证Footer更新**
   - 滚动到页面底部
   - ✅ 网站名称应该更新
   - ✅ 网站描述应该更新
   - ✅ 联系邮箱应该更新（可点击测试）
   - ✅ 联系电话应该更新（可点击测试）
   - ✅ 版权信息中的网站名称应该更新

4. **检查控制台日志**
   - 应该看到：`✅ 网站设置已加载: {siteName: "新名称", ...}`

### 步骤 4：测试其他页面

1. **测试律师列表页**
   ```
   http://localhost:3000/lawyers
   ```
   - Header和Footer应该显示新设置

2. **测试模板页面**
   ```
   http://localhost:3000/templates
   ```
   - Header和Footer应该显示新设置

3. **测试关于我们页面**
   ```
   http://localhost:3000/about
   ```
   - Header和Footer应该显示新设置

## 🐛 常见问题排查

### 问题1：修改后没有看到变化

**解决方案：**
1. 硬刷新浏览器（Ctrl+Shift+R）
2. 清除浏览器缓存
3. 检查控制台是否有错误
4. 确认数据库中的设置已更新

**验证数据库：**
```sql
SELECT * FROM settings WHERE key = 'site';
```

### 问题2：控制台显示错误

**可能的错误：**
- `Error loading site name` - 数据库连接问题
- `无法读取文件` - 文件路径问题
- `RLS policy violation` - 权限问题

**解决方案：**
1. 检查 `.env.local` 中的Supabase配置
2. 确认settings表存在且有数据
3. 检查RLS策略是否正确

### 问题3：Admin页面无法保存

**检查项：**
1. 是否已登录Admin账号
2. 数据库连接是否正常
3. 控制台是否有错误信息
4. 网络请求是否成功（Network标签）
