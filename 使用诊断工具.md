## 根据诊断结果解决问题

### 问题 1：用户不是管理员

**诊断结果显示**：
```
用户类型: user
是否管理员: 否
```

**解决方法**：
1. 访问 Supabase SQL Editor：
   https://supabase.com/dashboard/project/ovtrvzbftinsfwytzgwy/sql/new

2. 执行以下 SQL：
```sql
UPDATE profiles 
SET user_type = 'admin' 
WHERE email = 'test@example.com';
```

3. 刷新诊断页面，重新运行诊断

### 问题 3：保存权限被拒绝

**诊断结果显示**：
```
❌ 保存测试失败: new row violates row-level security policy
错误代码: 42501
```

**解决方法**：
1. 确认用户是管理员（见问题 1）
2. 检查 RLS 策略是否正确
3. 在 Supabase SQL Editor 执行：

```sql
-- 删除旧策略
DROP POLICY IF EXISTS "Only admins can update settings" ON settings;
DROP POLICY IF EXISTS "Only admins can insert settings" ON settings;

-- 重新创建策略
CREATE POLICY "Only admins can update settings"
  ON settings FOR UPDATE
  USING (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE id = auth.uid() AND user_type = 'admin'
    )
  );

CREATE POLICY "Only admins can insert settings"
  ON settings FOR INSERT
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE id = auth.uid() AND user_type = 'admin'
    )
  );
```
