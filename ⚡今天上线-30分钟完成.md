## 🚀 步骤 1：部署到 Vercel（10 分钟）

### 方法 A：使用 Vercel CLI（最快）

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel

# 4. 配置环境变量（在命令行提示时输入）
# 或者在 Vercel 网站配置

# 5. 部署到生产环境
vercel --prod
```

### 方法 B：使用 Vercel 网站（推荐）

1. 访问 [vercel.com](https://vercel.com)
2. 使用 GitHub 登录
3. 点击 "Add New Project"
4. 选择您的 GitHub 仓库
5. 配置环境变量：
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJxxx...
   SUPABASE_SERVICE_ROLE_KEY=eyJxxx...
   NEXT_PUBLIC_SITE_URL=https://your-project.vercel.app
   NODE_ENV=production
   ```
6. 点击 "Deploy"
7. 等待 2-3 分钟

**完成！** 您的网站已上线：`https://your-project.vercel.app`

## 🌐 步骤 2：配置自定义域名（10 分钟）

### 如果您已有域名

#### 在 Vercel 中添加域名
1. 进入 Vercel 项目设置
2. 点击 "Domains"
3. 添加您的域名：`legalmy.com`
4. Vercel 会显示需要配置的 DNS 记录

#### 在域名提供商配置 DNS
添加 CNAME 记录：
```
Type: CNAME
Name: @
Value: cname.vercel-dns.com

Type: CNAME
Name: www
Value: cname.vercel-dns.com
```

#### 等待 DNS 生效
- 通常 5-30 分钟
- 最长可能需要 24 小时

### 如果您还没有域名

**暂时使用 Vercel 提供的域名：**
```
https://your-project.vercel.app
```

**稍后购买域名：**
- Namecheap
- Cloudflare Registrar
- GoDaddy

## 🔒 步骤 3：配置 Cloudflare（5 分钟）

### 如果使用自定义域名

1. 注册 Cloudflare 账号
2. 添加您的域名
3. 修改域名 NS 记录指向 Cloudflare
4. 在 Cloudflare 配置：
   - SSL/TLS：Full (strict)
   - Always Use HTTPS：开启
   - Auto Minify：开启

### 如果暂时使用 Vercel 域名

**跳过此步骤**，Vercel 已自动提供 HTTPS。

## 🆘 遇到问题？

### 部署失败
1. 检查环境变量是否正确
2. 检查 Supabase 连接是否正常
3. 查看 Vercel 部署日志

### 域名无法访问
1. 检查 DNS 配置是否正确
2. 等待 DNS 生效（最长 24 小时）
3. 清除浏览器缓存

### 数据库连接失败
1. 检查 Supabase URL 是否正确
2. 检查 API Key 是否正确
3. 检查 Supabase 项目是否暂停
