## 📦 方案 A：Vercel 部署（详细步骤）

### 第一步：准备 Git 仓库

#### 1.1 初始化 Git（如果还没有）
```bash
git init
git add .
git commit -m "Initial commit - Ready for deployment"
```

#### 1.2 创建 GitHub 仓库
1. 访问 https://github.com/new
2. 创建新仓库（可以是私有仓库）
3. 不要初始化 README

#### 1.3 推送代码
```bash
git remote add origin https://github.com/你的用户名/你的仓库名.git
git branch -M main
git push -u origin main
```

### 第二步：配置环境变量

#### 2.1 创建生产环境配置
创建 `.env.production` 文件：

```env
# Supabase 配置（生产环境）
NEXT_PUBLIC_SUPABASE_URL=你的生产环境URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的生产环境密钥

# 网站配置
NEXT_PUBLIC_SITE_URL=https://你的域名.com

# 其他配置...
```

⚠️ **重要：** 不要将 `.env.production` 提交到 Git！

#### 2.2 确认 .gitignore
确保 `.gitignore` 包含：
```
.env
.env.local
.env.production
.env.development
```

### 第三步：部署到 Vercel

#### 3.1 注册 Vercel 账号
访问：https://vercel.com/signup

使用 GitHub 账号登录（推荐）

#### 3.2 导入项目
1. 点击 "Add New..." → "Project"
2. 选择您的 GitHub 仓库
3. 点击 "Import"

#### 3.3 配置项目
**Framework Preset:** Next.js（自动检测）

**Root Directory:** `./`（默认）

**Build Command:** `npm run build`（默认）

**Output Directory:** `.next`（默认）

**Install Command:** `npm install`（默认）

#### 3.4 配置环境变量
在 Vercel 项目设置中添加环境变量：

1. 点击 "Environment Variables"
2. 添加以下变量：

```
NEXT_PUBLIC_SUPABASE_URL = 你的生产环境URL
NEXT_PUBLIC_SUPABASE_ANON_KEY = 你的生产环境密钥
NEXT_PUBLIC_SITE_URL = https://你的域名.vercel.app
```

#### 3.5 部署
点击 "Deploy" 按钮

等待部署完成（通常 2-5 分钟）

### 第四步：配置生产数据库

#### 4.1 创建生产 Supabase 项目
1. 访问 https://supabase.com/dashboard
2. 创建新项目（选择合适的区域）
3. 等待项目初始化完成

#### 4.2 执行数据库迁移
在 Supabase SQL Editor 中依次执行：

```sql
-- 1. 基础表
001_create_tenants_table.sql
002_create_tenant_settings_table.sql
003_create_audit_logs_table.sql
004_create_system_settings_table.sql
005_create_password_reset_tokens_table.sql

-- 2. 修改表结构
006_add_tenant_columns.sql

-- 3. 权限和策略
007_create_rls_policies.sql
008_create_helper_functions.sql
009_set_tenant_id_not_null.sql

-- 4. 修复和优化
010_fix_rls_infinite_recursion_v2.sql

-- 5. 设置表（重要！）
011_create_settings_table.sql
```

#### 4.3 创建超级管理员
执行 `010_create_super_admin.sql` 或手动创建

#### 4.4 配置存储桶
执行 `create-storage-bucket.sql`

### 第五步：验证部署

#### 5.1 访问网站
访问：`https://你的项目名.vercel.app`

#### 5.2 测试功能
- [ ] 首页正常显示
- [ ] 网站设置同步正常
- [ ] Admin 登录正常
- [ ] Super Admin 登录正常
- [ ] 数据库连接正常
- [ ] 图片上传正常

#### 5.3 检查控制台
按 F12 查看是否有错误

### 第六步：配置自定义域名（可选）

#### 6.1 在 Vercel 添加域名
1. 项目设置 → Domains
2. 添加您的域名
3. 按照提示配置 DNS

#### 6.2 配置 DNS
在您的域名提供商处添加记录：

**A 记录：**
```
@ → 76.76.21.21
```

**CNAME 记录：**
```
www → cname.vercel-dns.com
```

#### 6.3 等待生效
通常需要 5-30 分钟

## 🔧 部署后配置

### 1. 更新网站设置
访问：`https://你的域名.com/admin/settings`

更新：
- 网站名称
- 网站描述
- 联系邮箱
- 联系电话

### 2. 配置 SEO
更新 `src/app/layout.tsx` 中的 metadata：

```typescript
export const metadata: Metadata = {
  title: "您的网站名称",
  description: "您的网站描述",
  keywords: "关键词1, 关键词2",
  openGraph: {
    title: "您的网站名称",
    description: "您的网站描述",
    url: "https://您的域名.com",
    siteName: "您的网站名称",
    images: [
      {
        url: "https://您的域名.com/og-image.jpg",
        width: 1200,
        height: 630,
      },
    ],
  },
};
```

### 3. 配置 Google Analytics（可选）
在 `src/app/layout.tsx` 添加：

```typescript
<Script
  src="https://www.googletagmanager.com/gtag/js?id=GA_MEASUREMENT_ID"
  strategy="afterInteractive"
/>
<Script id="google-analytics" strategy="afterInteractive">
  {`
    window.dataLayer = window.dataLayer || [];
    function gtag(){dataLayer.push(arguments);}
    gtag('js', new Date());
    gtag('config', 'GA_MEASUREMENT_ID');
  `}
</Script>
```

### 4. 配置 Sitemap
创建 `src/app/sitemap.ts`：

```typescript
import { MetadataRoute } from 'next';

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: 'https://您的域名.com',
      lastModified: new Date(),
      changeFrequency: 'daily',
      priority: 1,
    },
    {
      url: 'https://您的域名.com/lawyers',
      lastModified: new Date(),
      changeFrequency: 'daily',
      priority: 0.8,
    },
    // 添加更多页面...
  ];
}
```

### 5. 配置 robots.txt
创建 `public/robots.txt`：

```
User-agent: *
Allow: /

Sitemap: https://您的域名.com/sitemap.xml
```
