## 🔧 故障排除

### 问题 1：API 返回 404

**原因**：数据库中没有租户

**解决方案**：
1. 在 Supabase SQL Editor 中执行：
```sql
SELECT * FROM tenants WHERE status = 'active';
```
2. 如果没有租户，执行：
```sql
INSERT INTO tenants (name, subdomain, status)
VALUES ('默认律所', 'default', 'active');
```

### 问题 2：前台显示默认价格（RM 99）

**原因**：
1. API 请求失败
2. 数据库中没有价格配置
3. 价格配置格式不正确

**解决方案**：
1. 打开浏览器开发者工具（F12）
2. 查看 Console 是否有错误
3. 查看 Network 标签，检查 `/api/public/tenant-config` 请求
4. 在 Admin 后台重新保存价格配置

### 问题 3：价格显示为 "RM 0"

**原因**：您在 Admin 后台设置的价格确实是 0

**解决方案**：
1. 访问 Admin 后台：`http://localhost:3000/admin/pricing`
2. 修改价格为非零值
3. 点击"保存设置"
4. 刷新前台页面

## 📝 注意事项

### 缓存

API 响应包含缓存头：
```
Cache-Control: public, s-maxage=300, stale-while-revalidate=600
```

这意味着：
- 浏览器会缓存 5 分钟
- 如果您修改了价格，可能需要等待 5 分钟或强制刷新（Ctrl + Shift + R）

### 性能

- API 调用在客户端进行（`useEffect`）
- 每次访问首页都会调用一次 API
- 未来可以考虑使用 Server Components 或 SSR 来提高性能

### 安全

- API 是公开的，任何人都可以访问
- 不包含敏感信息（只有价格、品牌等公开配置）
- 不需要身份验证
