# 超级管理员系统 - 安全验证说明

## 🔐 权限隔离机制

### 系统架构
本系统采用**三层权限架构**：

1. **超级管理员系统** (`/super-admin`)
   - 只有 `super_admin = true` 的用户可以访问
   - 可以管理所有租户和用户
   - 不属于任何租户（`tenant_id = NULL`）

2. **租户管理系统** (`/admin`)
   - 租户的管理员账号（`tenant_id` 不为空）
   - 只能管理自己租户的数据
   - 不能访问超级管理员系统

3. **租户子账号**
   - 租户下的普通用户（有 `parent_user_id`）
   - 只能访问租户系统的部分功能
   - 不能访问超级管理员系统

## ✅ 安全验证机制

### 1. 服务器端验证（Server-Side）

**位置**: `src/app/super-admin/(auth)/layout.tsx`

```typescript
// 第一层：验证用户是否登录
if (!session) {
  redirect('/super-admin/login');
}

// 第二层：验证用户是否是超级管理员
const { data: profile } = await supabase
  .from('profiles')
  .select('*')
  .eq('id', session.user.id)
  .maybeSingle();

if (!profile?.super_admin) {
  redirect('/super-admin/login');
}
```

**特点**：
- ✅ 在服务器端执行，无法被客户端绕过
- ✅ 每次页面加载都会验证
- ✅ 验证失败立即重定向到登录页

---

### 2. 客户端验证（Client-Side）

**位置**: `src/lib/auth/withSuperAdminAuth.tsx`

```typescript
// 检查用户认证状态
const { data: { user }, error: authError } = await supabase.auth.getUser();

if (authError || !user) {
  router.push('/super-admin/login');
  return;
}

// 检查 Super Admin 权限
const { data: profile, error: profileError } = await supabase
  .from('profiles')
  .select('super_admin, user_type, tenant_id')
  .eq('id', user.id)
  .single();

if (!profile?.super_admin) {
  router.push('/super-admin/login?error=unauthorized');
  return;
}
```

**特点**：
- ✅ 用于客户端组件的权限验证
- ✅ 实时检查权限状态
- ✅ 显示加载状态，提升用户体验

## 🚫 租户账号无法访问超级系统

### 验证流程

当租户管理员或子账号尝试访问超级管理员系统时：

```
1. 用户访问 /super-admin/*
   ↓
2. 服务器端 Layout 检查 session
   ✅ 用户已登录
   ↓
3. 查询 profiles 表
   ✅ profile 存在
   ↓
4. 检查 super_admin 字段
   ❌ super_admin = false (或 NULL)
   ↓
5. 立即重定向到 /super-admin/login
   ❌ 访问被拒绝
```

### 数据库层面的隔离

**profiles 表结构**：
```sql
CREATE TABLE profiles (
  id UUID PRIMARY KEY,
  email TEXT,
  tenant_id UUID,           -- 租户ID（超级管理员为 NULL）
  super_admin BOOLEAN,      -- 超级管理员标识
  role TEXT,                -- 角色（owner/admin/manager/user）
  parent_user_id UUID,      -- 父账号ID（子账号才有）
  ...
);
```

**权限判断逻辑**：
- **超级管理员**: `super_admin = true` AND `tenant_id IS NULL`
- **租户管理员**: `super_admin = false` AND `tenant_id IS NOT NULL` AND `parent_user_id IS NULL`
- **租户子账号**: `super_admin = false` AND `tenant_id IS NOT NULL` AND `parent_user_id IS NOT NULL`

## 🔒 多层防护机制

### 1. 路由层防护
- 超级管理员路由：`/super-admin/*`
- 租户管理路由：`/admin/*`
- 完全隔离，不同的 Layout 和权限验证

### 2. API 层防护
所有超级管理员 API 都应该验证 `super_admin` 权限：

```typescript
// 示例：/api/super-admin/users/route.ts
const { data: profile } = await supabase
  .from('profiles')
  .select('super_admin')
  .eq('id', session.user.id)
  .single();

if (!profile?.super_admin) {
  return NextResponse.json({ error: '权限不足' }, { status: 403 });
}
```

### 3. 数据库层防护（RLS）
Row Level Security 策略确保：
- 租户只能访问自己的数据
- 超级管理员可以访问所有数据
- 子账号只能访问父账号授权的数据
