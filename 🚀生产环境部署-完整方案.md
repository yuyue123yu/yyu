## 🎯 推荐方案：Vercel（最简单）

### 为什么选择 Vercel？
- ✅ **完全免费**（个人项目）
- ✅ **零配置**，3 分钟部署
- ✅ **永不关闭**，24/7 运行
- ✅ **自动 HTTPS**
- ✅ **全球 CDN**，访问速度快
- ✅ **自动扩展**，流量大时自动增加资源

### 快速部署（3 分钟）

#### Windows PowerShell:
```powershell
# 运行部署脚本
.\deploy.ps1

# 选择选项 1（Vercel 部署）
```

#### 或手动部署:
```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录 Vercel
vercel login

# 3. 部署到生产环境
vercel --prod
```

### 部署后
- Vercel 会给您一个域名：`your-project.vercel.app`
- 可以绑定自定义域名
- 每次推送代码到 GitHub，自动部署

## 🔧 方案 3：PM2 部署

### 适用场景
- 传统 VPS 服务器
- 需要进程管理
- 需要负载均衡

### 快速部署

#### Windows PowerShell:
```powershell
# 运行部署脚本
.\deploy.ps1

# 选择选项 2（PM2 部署）
```

#### 或手动部署:
```bash
# 1. 安装 PM2
npm install -g pm2

# 2. 构建项目
npm run build

# 3. 启动应用
pm2 start ecosystem.config.js

# 4. 保存配置
pm2 save

# 5. 设置开机自启
pm2 startup
```

### PM2 常用命令
```bash
pm2 status              # 查看状态
pm2 logs                # 查看日志
pm2 restart malai-legal # 重启应用
pm2 stop malai-legal    # 停止应用
pm2 monit               # 监控面板
```

## 🔧 常见问题

### Q1: 为什么开发服务器会自动关闭？
**A**: `npm run dev` 是前台进程，关闭终端就停止。生产环境使用后台进程或托管平台，不会关闭。

### Q2: 如何确保服务器永不关闭？
**A**: 
- **Vercel**: 自动管理，无需担心
- **Docker**: 使用 `restart: always`
- **PM2**: 使用 `autorestart: true` + `pm2 startup`

### Q3: 服务器崩溃怎么办？
**A**: 
- **Vercel**: 自动恢复
- **Docker**: 自动重启容器
- **PM2**: 自动重启进程

### Q4: 如何查看日志？
**A**: 
- **Vercel**: 在 Vercel 控制台查看
- **Docker**: `docker-compose logs -f`
- **PM2**: `pm2 logs`

### Q5: 如何更新代码？
**A**: 
- **Vercel**: 推送代码到 GitHub，自动部署
- **Docker**: 重新构建镜像：`docker-compose up -d --build`
- **PM2**: 重新构建并重启：`npm run build && pm2 restart malai-legal`
