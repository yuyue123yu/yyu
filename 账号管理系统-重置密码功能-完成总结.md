- 密码仅显示一次
- 管理员可以一键复制密码
- 适合需要立即提供密码的情况

## 🔐 安全特性

### 权限验证
- 管理员可以重置任何用户的密码
- 租户隔离：非 super_admin 只能重置自己租户的用户
- owner 不能重置其他 owner 的密码（除非是 super_admin）

### 审计日志
- 记录所有密码重置操作
- 包含操作人、目标用户、重置方式、IP地址、User Agent
- 存储在 `password_reset_history` 表和 `audit_logs` 表

### 密码安全
- 临时密码长度：12位
- 包含大小写字母、数字、特殊字符
- 随机生成，打乱顺序
- 仅显示一次，不可再次查看

## 📝 使用流程

### 管理员重置用户密码
1. 进入"用户管理"或"子账号管理"页面
2. 点击用户行的"钥匙"图标
3. 弹出重置密码模态框
4. 选择重置方式：
   - **邮件方式**：点击"确认重置" → 显示成功提示 → 自动关闭
   - **临时密码方式**：点击"确认重置" → 显示临时密码 → 复制密码 → 点击"完成"
5. 将密码发送给用户

### 用户修改自己的密码
1. 点击侧边栏"系统设置"
2. 在"安全设置"卡片右上角点击"修改密码"按钮
3. 跳转到 `/settings/security` 页面
4. 输入当前密码
5. 输入新密码（查看密码强度指示器）
6. 确认新密码
7. 点击"修改密码"按钮
8. 显示成功提示
