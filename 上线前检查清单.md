## ✅ 账号切换安全性检查

### 当前实现的安全机制

#### 1. **登录前自动登出**
```typescript
// src/app/admin/login/page.tsx
await supabase.auth.signOut(); // 登录前先登出，确保账号切换安全
```
✅ **作用**：防止旧账号的 Session 残留

#### 2. **服务端权限验证**
```typescript
// src/app/admin/layout.tsx
const { data: { session } } = await supabase.auth.getSession();
if (!session) redirect('/admin/login');

const { data: profile } = await supabase.from('profiles')...
if (profile.user_type !== 'admin' && !profile.super_admin) {
  redirect('/admin/login');
}
```
✅ **作用**：每次访问都验证权限，防止未授权访问

#### 3. **HTTP-Only Cookie**
```typescript
// src/app/api/auth/callback/route.ts
res.cookies.set('sb-access-token', access_token, {
  httpOnly: true,  // 防止 XSS 攻击
  sameSite: 'lax', // 防止 CSRF 攻击
  secure: true,    // 生产环境强制 HTTPS
});
```
✅ **作用**：安全存储认证信息

#### 4. **RLS 数据隔离**
- 每个租户的数据通过 `tenant_id` 隔离
- Row Level Security 策略确保用户只能访问自己租户的数据
✅ **作用**：防止跨租户数据泄露

```sql
-- 1. 创建测试账号 A（如果不存在）
-- 在 Supabase Auth 中创建

-- 2. 创建测试账号 B（如果不存在）
-- 在 Supabase Auth 中创建

-- 3. 验证两个账号的租户隔离
SELECT 
  p.id,
  p.email,
  p.tenant_id,
  t.name as tenant_name
FROM profiles p
LEFT JOIN tenants t ON p.tenant_id = t.id
WHERE p.email IN ('账号A邮箱', '账号B邮箱');

-- 4. 验证服务数据隔离
SELECT 
  s.id,
  s.name_tc,
  s.tenant_id,
  t.name as tenant_name
FROM services s
LEFT JOIN tenants t ON s.tenant_id = t.id
ORDER BY s.tenant_id, s.display_order;
```
