### 场景 B: 部署仍然失败 ❌

需要检查新的构建日志，可能的问题：

#### B1. Event handlers 错误仍然存在

**定位方法**:

```bash
# 本地运行以获取详细错误
NEXT_RUNTIME_LOG=1 npm run build
```

**修复方法**:

- 找到传递 `onClick` 等函数给 Client Component 的地方
- 将父组件改为 Client Component（添加 `'use client'`）
- 或将事件逻辑移到 Client Component 内部

#### B2. Server Component 调用自身 API

**搜索命令**:

```bash
# 搜索可能的问题
grep -r "fetch('/api/" src/
grep -r 'fetch("/api/' src/
grep -r "NEXT_PUBLIC_URL" src/
```

**修复方法**:

- 将 `fetch('/api/...')` 改为直接调用 service/db
- 例如：

  ```typescript
  // 错误
  const res = await fetch('/api/public/services')

  // 正确
  import { getServices } from '@/lib/services'
  const services = await getServices()
  ```

#### B3. cookies() 在 layout.tsx 中使用

**搜索命令**:

```bash
grep -r "cookies()" src/app/**/layout.tsx
```

**修复方法**:

- 在 layout.tsx 顶部添加：
  ```typescript
  export const dynamic = 'force-dynamic'
  ```
- 或将 cookies() 调用移出 layout

## 💡 关键提示

### 为什么本地构建成功，Vercel 失败？

1. **环境差异**: Vercel 使用冷启动、沙盒环境、并发限制
2. **超时限制**: Vercel 有严格的 60 秒超时（已增加到 180 秒）
3. **问题放大**: 本地隐藏的问题在 Vercel 会被放大

### 当前策略：先 SSR，后优化

- ✅ 当前目标：稳定部署（使用 SSR）
- ⏳ 后续优化：逐步添加 SSG、ISR、缓存
- ❌ 不要：过早优化导致部署失败

### 部署时间预期

- **正常部署**: 2-3 分钟
- **首次部署**: 可能需要 3-5 分钟
- **超过 5 分钟**: 可能有问题，需要查看日志
