## 🎯 核心问题分析

### 问题 1：登录成功但跳转失败

**可能原因**：
1. **Session 未正确保存**
   - Supabase 客户端配置问题
   - Cookie 设置问题
   - 浏览器缓存问题

2. **跳转被拦截**
   - Layout 中的权限检查
   - Middleware 拦截
   - HOC (withSuperAdminAuth) 阻止

3. **RLS 策略问题**
   - Profile 查询失败
   - 权限验证失败

### 问题 2：Dashboard 访问被重定向

**原因**：
- `/admin` 的 layout 使用 `useAuth()` 检查登录状态
- 如果 `user` 为 null，自动重定向到 `/admin/login`
- `/super-admin` 使用 `withSuperAdminAuth` HOC 检查权限

## 💡 建议的调试步骤

### 步骤 1：验证登录状态

在登录成功后，在浏览器控制台执行：
```javascript
const supabase = createClient();
const { data: { session } } = await supabase.auth.getSession();
console.log('Session:', session);
console.log('User:', session?.user);
```

**预期结果**：
- `session` 不为 null
- `session.user` 包含用户信息

### 步骤 2：验证 Profile 查询

```javascript
const supabase = createClient();
const { data: { user } } = await supabase.auth.getUser();
const { data: profile, error } = await supabase
  .from('profiles')
  .select('*')
  .eq('id', user.id)
  .maybeSingle();

console.log('Profile:', profile);
console.log('Error:', error);
```

**预期结果**：
- `profile` 不为 null
- `error` 为 null

### 步骤 3：检查 Cookie

在浏览器开发者工具中：
1. 打开 Application/Storage 标签
2. 查看 Cookies
3. 查找 `sb-` 开头的 Cookie

**预期结果**：
- 应该有 `sb-<project-ref>-auth-token` Cookie
- Cookie 应该包含 access_token 和 refresh_token

### 步骤 4：检查网络请求

在 Network 标签中：
1. 执行登录
2. 查看 `/auth/v1/token?grant_type=password` 请求
3. 检查响应

**预期结果**：
- 状态码 200
- 响应包含 access_token 和 user 信息
