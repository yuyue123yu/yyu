## 🚀 部署步骤

### 步骤 1: 执行数据库迁移

在 Supabase Dashboard 的 SQL Editor 中执行：

```sql
-- 执行文件: supabase/013_create_tenant_storage.sql
```

这将创建：
- `tenant-assets` 存储桶
- 文件上传权限策略
- 文件访问权限策略

### 步骤 2: 配置 Supabase 环境变量

确保 `.env.local` 中有以下配置：

```env
NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

### 步骤 3: 重启开发服务器

```bash
npm run dev
```

### 步骤 4: 测试功能

1. 登录租户管理后台 `/admin`
2. 点击侧边栏的"品牌设置"
3. 上传 Logo
4. 选择主题颜色
5. 填写公司信息
6. 点击"显示预览"查看效果
7. 点击"保存设置"

## 🐛 常见问题

### Q: Logo 上传失败？
A: 检查以下几点：
1. 文件格式是否支持（JPG, PNG, SVG, WebP, GIF）
2. 文件大小是否超过 5MB
3. Supabase Storage Bucket 是否已创建
4. 权限策略是否正确配置

### Q: 颜色不生效？
A: 确保：
1. 颜色格式为 HEX（如 #1E40AF）
2. 已点击"保存设置"按钮
3. 前台页面已刷新

### Q: 没有权限修改？
A: 检查：
1. 用户角色是否为 Owner 或 Admin
2. 用户是否关联了租户（tenant_id）
3. 是否已登录
