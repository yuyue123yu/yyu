# 價格管理優化方案 🎯

## 問題分析

### 當前架構的問題
1. **價格配置過於集中**：所有價格都在一個「價格配置」頁面設置
2. **缺乏靈活性**：無法為不同服務、律師、模板設置獨立價格
3. **維護困難**：修改價格需要在多個地方同步
4. **擴展性差**：無法支持複雜的定價策略（如折扣、套餐等）

### 用戶的建議 ✅
**將價格配置集成到各個功能板塊中**，讓每個實體（服務、律師、模板）都有自己的價格設置。

---

## 優化方案

### 方案 A：分散式價格管理（推薦）⭐

#### 1. 服務管理（Services Management）

**數據庫表：`services`**
```sql
CREATE TABLE services (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  tenant_id UUID NOT NULL REFERENCES tenants(id),
  name_zh TEXT NOT NULL,           -- 服務名稱（中文）
  name_en TEXT,                     -- 服務名稱（英文）
  description_zh TEXT,              -- 服務描述（中文）
  description_en TEXT,              -- 服務描述（英文）
  category TEXT NOT NULL,           -- 類別（debt, family, business等）
  icon TEXT,                        -- 圖標
  base_price DECIMAL(10,2),        -- 基礎價格
  premium_price DECIMAL(10,2),     -- 高級價格
  currency TEXT DEFAULT 'MYR',     -- 貨幣
  duration_minutes INTEGER,         -- 服務時長（分鐘）
  is_active BOOLEAN DEFAULT true,   -- 是否啟用
  display_order INTEGER DEFAULT 0,  -- 顯示順序
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Admin 管理界面：`/admin/services`**
- 列表顯示所有服務
- 可以添加、編輯、刪除服務
- 每個服務可以設置獨立的價格
- 支持拖拽排序

#### 2. 律師管理（Lawyers Management）

**數據庫表：`lawyers`**
```sql
CREATE TABLE lawyers (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  tenant_id UUID NOT NULL REFERENCES tenants(id),
  user_id UUID REFERENCES profiles(id),
  name TEXT NOT NULL,
  specialty TEXT[],                 -- 專長領域（數組）
  bio TEXT,                         -- 個人簡介
  education TEXT,                   -- 教育背景
  experience_years INTEGER,         -- 執業年限
  consultation_price DECIMAL(10,2), -- 咨詢價格（每小時）
  case_price DECIMAL(10,2),        -- 案件價格（固定費用）
  currency TEXT DEFAULT 'MYR',
  response_time_minutes INTEGER,    -- 平均響應時間
  success_rate DECIMAL(5,2),       -- 成功率（%）
  total_cases INTEGER DEFAULT 0,    -- 處理案件數
  rating DECIMAL(3,2),             -- 評分（0-5）
  is_online BOOLEAN DEFAULT false,  -- 是否在線
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Admin 管理界面：`/admin/lawyers`**
- 列表顯示所有律師
- 可以添加、編輯、刪除律師
- 每個律師可以設置獨立的咨詢價格和案件價格
- 顯示律師的統計數據（案件數、評分等）

#### 3. 模板管理（Templates Management）

**數據庫表：`templates`**
```sql
CREATE TABLE templates (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  tenant_id UUID NOT NULL REFERENCES tenants(id),
  name_zh TEXT NOT NULL,
  name_en TEXT,
  description_zh TEXT,
  description_en TEXT,
  category TEXT NOT NULL,           -- 類別
  file_url TEXT,                    -- 文件URL
  file_size INTEGER,                -- 文件大小（字節）
  file_format TEXT,                 -- 文件格式（PDF, DOCX等）
  price DECIMAL(10,2) DEFAULT 0,   -- 價格（0表示免費）
  currency TEXT DEFAULT 'MYR',
  download_count INTEGER DEFAULT 0, -- 下載次數
  is_free BOOLEAN DEFAULT true,     -- 是否免費
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Admin 管理界面：`/admin/templates`**
- 列表顯示所有模板
- 可以上傳、編輯、刪除模板
- 每個模板可以設置價格（免費或付費）
- 顯示下載統計

### 方案 B：混合式價格管理

保留「價格配置」頁面作為**默認價格設置**，但允許在各個功能板塊中**覆蓋默認價格**。

#### 優點
- 向後兼容，不需要大規模重構
- 可以快速設置全局默認價格
- 靈活性高，可以為特定實體設置獨立價格

#### 實現方式
1. **價格配置頁面**（`/admin/pricing`）
   - 設置默認的咨詢價格、審核價格等
   - 作為全局默認值

2. **服務管理頁面**（`/admin/services`）
   - 如果服務沒有設置價格，使用默認價格
   - 如果服務設置了價格，使用服務的價格（覆蓋默認）

3. **前台顯示邏輯**
   ```typescript
   // 優先級：服務價格 > 默認價格
   const price = service.base_price || tenantConfig.pricing.consultation_base_price;
   ```

## 建議

### 短期方案（1-2 週）
採用 **方案 B（混合式）**：
- 保留現有的「價格配置」頁面
- 在服務管理中添加價格覆蓋功能
- 前台優先使用服務價格，沒有則使用默認價格

### 長期方案（1-2 個月）
採用 **方案 A（分散式）**：
- 完全重構價格管理系統
- 為每個實體（服務、律師、模板）設置獨立價格
- 支持複雜的定價策略（折扣、套餐、會員價等）
