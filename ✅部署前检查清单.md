# ✅ 部署前检查清单

**使用说明**: 每次部署到生产环境前，请完成此清单的所有项目

---

## 📋 代码质量检查

### 本地测试
- [ ] **本地构建成功**: `npm run build`
- [ ] **类型检查通过**: `npx tsc --noEmit`
- [ ] **Lint 检查通过**: `npm run lint`
- [ ] **本地运行正常**: `npm run dev` 并测试主要功能

### 代码审查
- [ ] **代码已提交**: 所有更改已 commit
- [ ] **分支已推送**: `git push origin <branch>`
- [ ] **无调试代码**: 移除 `console.log`, `debugger` 等
- [ ] **无敏感信息**: 检查是否有硬编码的密钥、密码

---

## 🔐 环境变量检查

### Vercel 环境变量
访问: Vercel Dashboard > Settings > Environment Variables

- [ ] `NEXT_PUBLIC_SUPABASE_URL` - 已设置
- [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY` - 已设置
- [ ] `SUPABASE_SERVICE_ROLE_KEY` - 已设置
- [ ] `NEXT_PUBLIC_APP_URL` - 已设置为正确的域名

### 环境变量验证
```bash
# 在 Vercel Dashboard 检查
# 确保所有环境变量都已设置且正确
```

---

## 🗄️ 数据库检查

### Supabase 状态
访问: Supabase Dashboard

- [ ] **数据库在线**: Database 状态正常
- [ ] **RLS 策略启用**: 检查 Authentication > Policies
- [ ] **表结构正确**: 检查所有必需的表都存在
- [ ] **测试数据准备**: 至少有一个测试租户和用户

### 必需的表
- [ ] `profiles` - 用户表
- [ ] `tenants` - 租户表
- [ ] `tenant_settings` - 租户设置表
- [ ] `audit_logs` - 审计日志表

---

## 🔄 Git 版本管理

### 分支状态
- [ ] **当前分支**: 确认在正确的分支 (`main` 或 `production`)
- [ ] **代码已同步**: `git pull origin <branch>`
- [ ] **无未提交更改**: `git status` 显示干净
- [ ] **标签已创建**: 如果是重要版本，创建 Git 标签

### 版本标签 (可选)
```bash
# 创建版本标签
git tag v1.x.x -m "Release description"
git push origin v1.x.x
```

## 🚀 Vercel 部署检查

### 部署前
- [ ] **检查上次部署**: 确认上次部署成功
- [ ] **检查构建日志**: 无警告或错误
- [ ] **检查部署分支**: 确认 Vercel 监听正确的分支

### 部署配置
访问: Vercel Dashboard > Settings

- [ ] **Build Command**: `npm run build`
- [ ] **Output Directory**: `.next`
- [ ] **Install Command**: `npm install`
- [ ] **Node Version**: 18.x 或更高

## 🧪 功能测试

### 关键功能
- [ ] **首页加载**: 访问首页，检查是否正常显示
- [ ] **用户登录**: 测试登录功能
- [ ] **管理后台**: 测试 `/admin` 访问
- [ ] **超级管理员**: 测试 `/super-admin` 访问
- [ ] **API 端点**: 测试关键 API 是否响应

### 测试账号

- [ ] 普通用户账号
- [ ] 租户管理员账号
- [ ] 超级管理员账号 (`admin@example.com`)

## 📊 性能检查

### 构建输出
检查构建日志中的警告：
- [ ] **无 UTF-8 错误**: 检查构建日志
- [ ] **无类型错误**: TypeScript 编译成功
- [ ] **无 ESLint 错误**: Lint 检查通过
- [ ] **构建时间合理**: < 5 分钟

### 包大小
- [ ] **First Load JS**: < 200 KB (理想)
- [ ] **页面大小**: 检查是否有异常大的页面

## 🔒 安全检查

### 代码安全
- [ ] **无硬编码密钥**: 检查代码中是否有 API 密钥
- [ ] **环境变量使用**: 敏感信息都使用环境变量
- [ ] **RLS 策略**: 数据库 RLS 策略已启用
- [ ] **HTTPS 强制**: 生产环境强制使用 HTTPS

# 如果有高危漏洞，修复它们

## 🎯 部署后验证

### 立即检查 (部署后 5 分钟内)
- [ ] **网站可访问**: 访问生产 URL
- [ ] **首页正常**: 首页内容显示正确
- [ ] **登录功能**: 测试登录
- [ ] **API 响应**: 测试关键 API
- [ ] **无控制台错误**: 打开浏览器控制台，检查是否有错误

### 监控检查 (部署后 1 小时内)
- [ ] **Vercel 日志**: 检查是否有错误日志
- [ ] **Sentry 错误**: 检查 Sentry Dashboard (如已配置)
- [ ] **Supabase 日志**: 检查数据库连接和查询
- [ ] **性能指标**: 检查页面加载时间
