# 🚀 生产环境部署指南

## 问题：开发服务器自动关闭

**开发环境问题**：
- `npm run dev` 是开发服务器，关闭终端就停止
- 每次打开都需要手动启动
- 不适合生产环境使用

**生产环境解决方案**：
- 使用专业的托管平台或进程管理工具
- 服务器会持续运行，不会自动关闭
- 支持自动重启、负载均衡等功能

## 🌟 方案 1：Vercel 部署（最推荐）

### 优点
✅ **完全免费**（个人项目）
✅ **零配置**，自动构建和部署
✅ **全球 CDN**，访问速度快
✅ **自动 HTTPS**
✅ **Git 集成**，推送代码自动部署
✅ **永不关闭**，24/7 运行
✅ **自动扩展**，流量大时自动增加资源

### 部署步骤

#### 1. 安装 Vercel CLI
```bash
npm install -g vercel
```

#### 2. 登录 Vercel
```bash
vercel login
```

#### 3. 部署项目
```bash
# 在项目根目录执行
vercel

# 生产环境部署
vercel --prod
```

#### 4. 配置环境变量
在 Vercel 控制台添加环境变量：
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- 其他必要的环境变量

#### 5. 自动部署
- 连接 GitHub 仓库
- 每次推送代码，Vercel 自动部署
- 预览环境 + 生产环境

### 访问方式
- Vercel 会提供一个域名：`your-project.vercel.app`
- 可以绑定自定义域名

## 🐳 方案 2：Docker + 云服务器

### 优点
✅ **完全控制**服务器环境
✅ **可扩展**，适合大型项目
✅ **支持所有云平台**（阿里云、腾讯云、AWS 等）

### 部署步骤

#### 1. 创建 Dockerfile
```dockerfile
# Dockerfile
FROM node:18-alpine AS base

# 安装依赖
FROM base AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci

# 构建应用
FROM base AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# 生产环境
FROM base AS runner
WORKDIR /app

ENV NODE_ENV production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 3000

ENV PORT 3000

CMD ["node", "server.js"]
```

#### 2. 创建 docker-compose.yml
```yaml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_SUPABASE_URL=${NEXT_PUBLIC_SUPABASE_URL}
      - NEXT_PUBLIC_SUPABASE_ANON_KEY=${NEXT_PUBLIC_SUPABASE_ANON_KEY}
      - SUPABASE_SERVICE_ROLE_KEY=${SUPABASE_SERVICE_ROLE_KEY}
    restart: always
```

#### 3. 部署到服务器
```bash
# 构建镜像
docker-compose build

# 启动服务（后台运行，永不关闭）
docker-compose up -d

# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down
```

### 关键配置
- `restart: always` - 服务器重启后自动启动
- `-d` 参数 - 后台运行，不会因为关闭终端而停止

## 🔧 方案 3：PM2 进程管理（传统服务器）

### 优点
✅ **简单易用**
✅ **自动重启**（崩溃后自动恢复）
✅ **日志管理**
✅ **负载均衡**

### 部署步骤

#### 1. 安装 PM2
```bash
npm install -g pm2
```

#### 2. 构建生产版本
```bash
npm run build
```

#### 3. 创建 PM2 配置文件
```javascript
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'malai-legal',
    script: 'npm',
    args: 'start',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss',
    merge_logs: true,
    autorestart: true,
    max_memory_restart: '1G'
  }]
}
```

#### 4. 启动应用
```bash
# 启动应用（永不关闭）
pm2 start ecosystem.config.js

# 查看状态
pm2 status

# 查看日志
pm2 logs

# 重启应用
pm2 restart malai-legal

# 停止应用
pm2 stop malai-legal

# 开机自启动
pm2 startup
pm2 save
```

### 关键特性
- **自动重启**：应用崩溃后自动重启
- **开机自启**：服务器重启后自动启动应用
- **集群模式**：利用多核 CPU，提高性能
- **日志管理**：自动记录日志

## 🌐 方案 4：Nginx + PM2（完整方案）

### 架构
```
用户请求 → Nginx (反向代理) → PM2 (进程管理) → Next.js 应用
```

### Nginx 配置
```nginx
# /etc/nginx/sites-available/malai-legal
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 启用配置
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/malai-legal /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx

# 设置开机自启
sudo systemctl enable nginx
```

## 💡 常见问题

### Q1: 为什么开发服务器会自动关闭？
**A**: `npm run dev` 是前台进程，关闭终端就停止。生产环境使用后台进程或托管平台，不会关闭。

### Q2: 哪个方案最适合我？
**A**: 
- **个人项目/小型项目** → Vercel（免费、简单）
- **中型项目** → Docker + 云服务器
- **大型项目** → Nginx + PM2 + 负载均衡

### Q3: 如何确保服务器永不关闭？
**A**: 
- Vercel：自动管理，无需担心
- Docker：使用 `restart: always`
- PM2：使用 `autorestart: true` + `pm2 startup`

### Q4: 服务器崩溃怎么办？
**A**: 
- Vercel：自动恢复
- Docker：自动重启容器
- PM2：自动重启进程
