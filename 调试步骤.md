# 🔍 调试步骤 - 查找保存失败的原因

## 问题现状

您修改网站名称后点击保存，但前端和管理后台都没有更新。

---

## 步骤 1：查看浏览器 Console 错误

### 操作步骤

1. 打开浏览器开发者工具（按 F12）
2. 切换到 **Console** 标签
3. 清空 Console（点击 🚫 图标）
4. 访问系统设置页面：http://localhost:3000/admin/settings
5. 修改网站名称为"东南亚法律平台"
6. 点击"保存设置"
7. **查看 Console 中的输出**

### 预期输出

如果成功，应该看到：
```
开始保存设置...
✅ 网站设置保存成功
✅ 邮件设置保存成功
✅ 通知设置保存成功
✅ 安全设置保存成功
✅ 系统设置保存成功
🎉 所有设置保存成功！
```

### 可能的错误

#### 错误 1: "permission denied for table settings"
**原因**：测试用户不是管理员

**解决方法**：
```sql
-- 在 Supabase SQL Editor 执行
UPDATE profiles 
SET user_type = 'admin' 
WHERE email = 'test@example.com';
```

#### 错误 2: "relation 'settings' does not exist"
**原因**：settings 表未创建

**解决方法**：
在 Supabase SQL Editor 执行 `supabase/create-settings-table.sql` 中的 SQL

#### 错误 3: "new row violates row-level security policy"
**原因**：RLS 策略阻止了操作

**解决方法**：
```sql
-- 检查 RLS 策略
SELECT * FROM pg_policies WHERE tablename = 'settings';

-- 如果需要，重新创建策略（见下方）
```

## 步骤 4：检查 RLS 策略

### 在 Supabase SQL Editor 执行

```sql
-- 查看 settings 表的 RLS 策略
SELECT 
  schemaname,
  tablename,
  policyname,
  permissive,
  roles,
  cmd,
  qual,
  with_check
FROM pg_policies 
WHERE tablename = 'settings';
```

**预期结果**：
应该有 3 个策略：
1. `Settings are viewable by everyone` (SELECT)
2. `Only admins can update settings` (UPDATE)
3. `Only admins can insert settings` (INSERT)

### 如果策略缺失或错误

重新创建策略：
```sql
-- 删除旧策略
DROP POLICY IF EXISTS "Settings are viewable by everyone" ON settings;
DROP POLICY IF EXISTS "Only admins can update settings" ON settings;
DROP POLICY IF EXISTS "Only admins can insert settings" ON settings;

-- 创建新策略
CREATE POLICY "Settings are viewable by everyone"
  ON settings FOR SELECT
  USING (true);

CREATE POLICY "Only admins can update settings"
  ON settings FOR UPDATE
  USING (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE id = auth.uid() AND user_type = 'admin'
    )
  );

CREATE POLICY "Only admins can insert settings"
  ON settings FOR INSERT
  WITH CHECK (
    EXISTS (
      SELECT 1 FROM profiles
      WHERE id = auth.uid() AND user_type = 'admin'
    )
  );
```

## 步骤 5：手动测试保存

### 在浏览器 Console 执行

```javascript
// 1. 导入 Supabase 客户端
const { createClient } = await import('./src/lib/supabase/client.js');
const supabase = await createClient();

// 2. 测试保存网站设置
const { data, error } = await supabase
  .from('settings')
  .upsert({
    key: 'site',
    value: {
      siteName: '东南亚法律平台',
      siteDescription: '专业法律咨询平台',
      contactEmail: 'support@legalmy.com',
      contactPhone: '+60 3-1234 5678',
      defaultLanguage: 'zh'
    }
  }, { onConflict: 'key' });

// 3. 查看结果
if (error) {
  console.error("❌ 保存失败:", error);
} else {
  console.log("✅ 保存成功:", data);
}

// 4. 验证数据
const { data: checkData } = await supabase
  .from('settings')
  .select('*')
  .eq('key', 'site')
  .single();

console.log("数据库中的数据:", checkData);
```

## 常见问题和解决方案

### 问题 1：settings 表不存在

**解决方案**：
1. 访问 Supabase SQL Editor
2. 复制 `supabase/create-settings-table.sql` 的内容
3. 执行 SQL
4. 确认执行成功

### 问题 2：用户不是管理员

**解决方案**：
```sql
UPDATE profiles 
SET user_type = 'admin' 
WHERE email = 'test@example.com';
```

### 问题 3：RLS 策略阻止操作

**解决方案**：
1. 检查策略（见步骤 4）
2. 重新创建策略
3. 确认用户是管理员

### 问题 4：数据保存成功但前端不更新

**解决方案**：
1. 硬刷新浏览器（Ctrl + Shift + R）
2. 清除浏览器缓存
3. 检查 Header 和 Layout 组件的代码

## 快速诊断命令

在浏览器 Console 中运行：
```javascript
// 一键诊断
async function diagnose() {
  const { createClient } = await import('./src/lib/supabase/client.js');
  const supabase = await createClient();
  
  console.log("=== 诊断开始 ===\n");
  
  // 1. 检查用户
  const { data: { user } } = await supabase.auth.getUser();
  console.log("1. 当前用户:", user?.email);
  
  // 2. 检查权限
  const { data: profile } = await supabase
    .from('profiles')
    .select('user_type')
    .eq('id', user.id)
    .single();
  console.log("2. 用户类型:", profile?.user_type);
  
  // 3. 检查 settings 表
  const { data: settings, error } = await supabase
    .from('settings')
    .select('*');
  console.log("3. settings 表:", error ? `错误: ${error.message}` : `${settings.length} 条记录`);
  
  // 4. 测试保存
  const { error: saveError } = await supabase
    .from('settings')
    .upsert({
      key: 'test',
      value: { test: true }
    }, { onConflict: 'key' });
  console.log("4. 保存测试:", saveError ? `失败: ${saveError.message}` : "成功");
  
  // 清理测试数据
  await supabase.from('settings').delete().eq('key', 'test');
  
  console.log("\n=== 诊断完成 ===");
}

diagnose();
```
