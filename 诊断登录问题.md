# 🔍 Super Admin 登录问题诊断

## 问题描述
登录时显示：**"安全提示：登录功能已禁用，所有API接口已被禁用"**

## 可能的原因

### 1. 数据库中没有 Super Admin 用户
最可能的原因！

### 2. profiles 表不存在或缺少 super_admin 字段

### 3. Supabase 连接问题

### 4. RLS 策略阻止了查询

---

## 🔧 解决方案

### 方案 1：创建 Super Admin 用户（最可能）

#### 步骤 1：检查数据库

在 Supabase SQL Editor 执行：

```sql
-- 检查 profiles 表是否存在
SELECT * FROM profiles LIMIT 5;

-- 检查是否有 super_admin 用户
SELECT id, email, super_admin FROM profiles WHERE super_admin = true;
```

#### 步骤 2：创建 Super Admin 用户

如果没有 super_admin 用户，执行：

```sql
-- 方法 A：如果已有用户，升级为 super admin
UPDATE profiles 
SET super_admin = true 
WHERE email = '403940124@qq.com';

-- 方法 B：创建新的 super admin 用户
-- 首先在 Supabase Authentication 中创建用户
-- 然后执行：
INSERT INTO profiles (id, email, super_admin, created_at)
VALUES (
  '用户的UUID',  -- 从 auth.users 获取
  '403940124@qq.com',
  true,
  NOW()
)
ON CONFLICT (id) DO UPDATE SET super_admin = true;
```

### 方案 2：使用自动创建脚本

在 Supabase SQL Editor 执行：

```sql
-- 自动创建 Super Admin
DO $$
DECLARE
  user_id UUID;
BEGIN
  -- 查找用户
  SELECT id INTO user_id 
  FROM auth.users 
  WHERE email = '403940124@qq.com';
  
  IF user_id IS NOT NULL THEN
    -- 更新或插入 profile
    INSERT INTO profiles (id, email, super_admin, created_at)
    VALUES (user_id, '403940124@qq.com', true, NOW())
    ON CONFLICT (id) 
    DO UPDATE SET super_admin = true;
    
    RAISE NOTICE '✅ Super Admin 已创建/更新: %', user_id;
  ELSE
    RAISE NOTICE '❌ 未找到用户: 403940124@qq.com';
    RAISE NOTICE '请先在 Authentication 中创建此用户';
  END IF;
END $$;
```

### 方案 3：检查 RLS 策略

```sql
-- 检查 profiles 表的 RLS 策略
SELECT * FROM pg_policies WHERE tablename = 'profiles';

-- 临时禁用 RLS 测试（仅用于诊断）
ALTER TABLE profiles DISABLE ROW LEVEL SECURITY;

-- 测试后记得重新启用
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
```

### 2. 在 SQL Editor 执行

```sql
-- 一键修复脚本
DO $$
DECLARE
  user_id UUID;
  user_email TEXT := '403940124@qq.com';
BEGIN
  -- 1. 查找用户
  SELECT id INTO user_id FROM auth.users WHERE email = user_email;
  
  IF user_id IS NULL THEN
    RAISE EXCEPTION '❌ 用户不存在: %。请先在 Authentication 中创建用户', user_email;
  END IF;
  
  -- 2. 确保 profiles 表存在
  IF NOT EXISTS (SELECT 1 FROM information_schema.tables WHERE table_name = 'profiles') THEN
    RAISE EXCEPTION '❌ profiles 表不存在';
  END IF;
  
  -- 3. 创建或更新 profile
  INSERT INTO profiles (id, email, super_admin, user_type, created_at, updated_at)
  VALUES (user_id, user_email, true, 'super_admin', NOW(), NOW())
  ON CONFLICT (id) DO UPDATE 
  SET super_admin = true, 
      user_type = 'super_admin',
      updated_at = NOW();
  
  -- 4. 验证
  IF EXISTS (SELECT 1 FROM profiles WHERE id = user_id AND super_admin = true) THEN
    RAISE NOTICE '✅ Super Admin 创建成功！';
    RAISE NOTICE '   用户ID: %', user_id;
    RAISE NOTICE '   邮箱: %', user_email;
  ELSE
    RAISE EXCEPTION '❌ 创建失败';
  END IF;
END $$;
```
