### 5️⃣ 禁止 Server Component fetch 自己 API 🚨 关键

**这是你后面最容易再次踩坑的地方！**

#### ❌ 错误写法 (会导致死锁)

```typescript
// src/app/admin/users/page.tsx
export default async function UsersPage() {
  // ❌ 错误！Server Component fetch 自己的 API
  const response = await fetch('/api/tenant/users')
  const users = await response.json()
  
  return <UserList users={users} />
}
```

**为什么会死锁**:
1. Next.js 构建时尝试预渲染页面
2. Server Component 调用 `/api/tenant/users`
3. API 需要 `cookies()` 获取 session
4. 构建时没有 cookies
5. 💥 死锁 timeout

#### ✅ 正确写法

**方案 1: 直接调用数据库函数** (推荐)

```typescript
// src/lib/repositories/user.repo.ts
import { createClient } from '@/lib/supabase/server'

export async function getUsers(tenantId: string) {
  const supabase = await createClient()
  const { data, error } = await supabase
    .from('profiles')
    .select('*')
    .eq('tenant_id', tenantId)
  
  if (error) throw error
  return data
}

// src/app/admin/users/page.tsx
import { getUsers } from '@/lib/repositories/user.repo'

export default async function UsersPage() {
  // ✅ 正确！直接调用数据库
  const users = await getUsers('tenant-id')
  
  return <UserList users={users} />
}
```

**方案 2: 使用 Server Actions**

```typescript
// src/app/admin/users/actions.ts
'use server'

import { createClient } from '@/lib/supabase/server'

export async function getUsersAction() {
  const supabase = await createClient()
  // ... 数据库查询
}

// src/app/admin/users/page.tsx
import { getUsersAction } from './actions'

export default async function UsersPage() {
  const users = await getUsersAction()
  return <UserList users={users} />
}
```

**方案 3: Client Component + useEffect**

```typescript
// src/app/admin/users/page.tsx
'use client'

import { useEffect, useState } from 'react'

export default function UsersPage() {
  const [users, setUsers] = useState([])
  
  useEffect(() => {
    // ✅ 在客户端 fetch 是安全的
    fetch('/api/tenant/users')
      .then(res => res.json())
      .then(data => setUsers(data.users))
  }, [])
  
  return <UserList users={users} />
}
```

**总结**:
- ✅ Server Component → 直接调用数据库
- ✅ Client Component → fetch API
- ❌ Server Component → fetch 自己 API (死锁)

### 6️⃣ Middleware 不要做复杂逻辑 ⚠️ 重要

**当前状态**: Middleware 已临时禁用 (`matcher: []`)

**未来重新启用时的原则**:

#### ✅ Middleware 应该做的事

```typescript
// src/middleware.ts
export async function middleware(request: NextRequest) {
  // ✅ 简单的重定向
  if (request.nextUrl.pathname === '/old-path') {
    return NextResponse.redirect(new URL('/new-path', request.url))
  }
  
  // ✅ 基础的认证检查
  const token = request.cookies.get('auth-token')
  if (!token && request.nextUrl.pathname.startsWith('/admin')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
  
  // ✅ 添加简单的 header
  const response = NextResponse.next()
  response.headers.set('x-custom-header', 'value')
  return response
}
```

#### ❌ Middleware 不应该做的事

```typescript
// ❌ 不要在 middleware 里查询数据库
const { data: user } = await supabase
  .from('profiles')
  .select('*')
  .eq('id', userId)
  .single()

// ❌ 不要做复杂的权限检查
if (user.role === 'admin' && user.tenant.status === 'active') {
  // 复杂逻辑
}

// ❌ 不要做多次数据库查询
const tenant = await getTenant()
const permissions = await getPermissions()
const settings = await getSettings()

// ❌ 不要做复杂的 rewrite
if (complexCondition) {
  return NextResponse.rewrite(new URL('/complex-path', request.url))
}
```

**为什么**:
- Middleware 在每个请求时都运行
- 复杂逻辑会导致构建时死锁
- 数据库查询会拖慢所有请求

**正确做法**:
- 复杂逻辑 → 放在 Server Component
- 权限检查 → 放在 API Route Handler
- 数据库查询 → 放在 Server Actions

```typescript
// 1. Repository 层 (数据访问)
// src/repositories/user.repo.ts
export async function findUserById(id: string) {
  const supabase = await createClient()
  return supabase.from('profiles').select('*').eq('id', id).single()
}

// 2. Service 层 (业务逻辑)
// src/services/user.service.ts
export async function getUserWithPermissions(userId: string) {
  const user = await findUserById(userId)
  const permissions = await getPermissions(user.role)
  return { ...user, permissions }
}

// 3. API 层 (HTTP 接口)
// src/app/api/users/[id]/route.ts
export async function GET(request: NextRequest, { params }) {
  const user = await getUserWithPermissions(params.id)
  return NextResponse.json(user)
}

// 4. Page 层 (UI 展示)
// src/app/admin/users/[id]/page.tsx
export default async function UserDetailPage({ params }) {
  const user = await getUserWithPermissions(params.id)
  return <UserDetail user={user} />
}
```

### 配置 GitHub Actions

创建 `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  pull_request:
    branches: [main, production]
  push:
    branches: [main]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run linter
        run: npm run lint
      
      - name: Type check
        run: npm run type-check
      
      - name: Run tests (if exists)
        run: npm run test --if-present
      
      - name: Build
        run: npm run build
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
```

## 🚨 常见问题和解决方案

### 问题 1: 部署后出现 500 错误

**排查步骤**:
1. 检查 Vercel 日志: Dashboard > Deployments > Logs
2. 检查 Supabase 连接: 环境变量是否正确
3. 检查 RLS 策略: 是否阻止了数据访问
4. 检查 API 路由: 是否有语法错误

### 问题 2: 页面加载很慢

**优化方向**:
1. 检查是否所有页面都是 SSR (force-dynamic)
2. 优化数据库查询: 添加索引
3. 使用 ISR 代替 SSR
4. 启用 CDN 缓存

### 问题 3: 构建失败

**常见原因**:
1. TypeScript 类型错误
2. ESLint 错误
3. 环境变量缺失
4. 依赖版本冲突

**解决方法**:
```bash
# 本地测试构建
npm run build

# 检查类型
npm run type-check

# 检查 lint
npm run lint
```

### 问题 4: 微信无法打开

**原因**: 微信屏蔽了 vercel.app 域名

**解决方案**:
1. 购买自定义域名 (.com 约 $10/年)
2. 在 Vercel 绑定域名
3. 配置 DNS 记录
4. 更新环境变量 `NEXT_PUBLIC_APP_URL`
