# 服務管理系統 - 實施說明 📋

## 已完成的工作 ✅

### 1. 數據庫表（`services`）
**文件**: `supabase/011_create_services_table.sql`

**功能**:
- 存儲所有服務的完整信息
- 支持多語言（繁體中文、英文、馬來語）
- 包含價格設置、顯示設置、SEO 設置
- 自動插入 7 個默認服務（基於當前首頁）

**字段**:
- 基本信息：`name_zh`、`name_en`、`description_zh` 等
- 價格設置：`base_price`、`premium_price`、`currency`
- 顯示設置：`is_active`、`is_featured`、`badge`、`display_order`
- 統計數據：`case_count`、`view_count`
- SEO 設置：`seo_title_zh`、`seo_description_zh` 等

### 2. API 路由

#### Admin API（需要登錄）
- **GET** `/api/admin/services` - 獲取服務列表
- **POST** `/api/admin/services` - 創建新服務
- **GET** `/api/admin/services/[id]` - 獲取單個服務
- **PUT** `/api/admin/services/[id]` - 更新服務
- **DELETE** `/api/admin/services/[id]` - 刪除服務

#### 公開 API（不需要登錄）
- **GET** `/api/public/services` - 前台獲取服務列表

### 3. Admin 管理頁面
**文件**: `src/app/admin/services/page.tsx`

**功能**:
- 📋 服務列表顯示
- ➕ 添加新服務按鈕
- ✏️ 編輯服務
- 🗑️ 刪除服務
- 👁️ 啟用/停用服務
- 📊 統計信息（總服務數、啟用數、推薦數、總案件數）
- 🏷️ 標籤顯示（熱門、推薦、新品）

## 注意事項

1. **數據遷移**
   - 執行 SQL 文件前，請備份數據庫
   - 確保當前租戶狀態為 `active`

2. **RLS 策略**
   - 已配置租戶隔離
   - 公開 API 只返回啟用的服務

3. **緩存**
   - 公開 API 有 5 分鐘緩存
   - 修改服務後可能需要等待緩存過期
