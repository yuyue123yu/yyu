# ✅ 管理后台网站名称已修复

## 问题说明

您修改了网站名称为"东南亚法律平台"，但管理后台侧边栏仍然显示 "LegalMY Admin"。

**原因**：管理后台的布局文件 (`src/app/admin/layout.tsx`) 中的网站名称是硬编码的，没有从数据库读取。

---

## 修复内容

已修改 `src/app/admin/layout.tsx`：

### 1. 添加状态管理
```typescript
const [siteName, setSiteName] = useState("LegalMY");
```

### 2. 添加数据库读取函数
```typescript
const loadSiteName = async () => {
  const supabase = await createClient();
  const { data } = await supabase
    .from("settings")
    .select("value")
    .eq("key", "site")
    .single();

  if (data && data.value && data.value.siteName) {
    setSiteName(data.value.siteName);
  }
};
```

### 3. 页面加载时读取
```typescript
useEffect(() => {
  loadSiteName();
}, []);
```

### 4. 动态显示网站名称
```typescript
<Link href="/admin" className="text-xl font-bold text-primary-600">
  {siteName} Admin
</Link>
```

## 故障排查

### 问题 1：刷新后还是显示 "LegalMY Admin"

**可能原因**：
1. 数据库中的数据没有更新
2. 浏览器缓存
3. 数据库连接失败

**解决方法**：
1. 硬刷新浏览器（Ctrl + Shift + R）
2. 检查浏览器 Console 是否有错误
3. 验证数据库中的数据（见上方）
4. 重新保存设置

### 问题 2：Console 显示错误

**打开浏览器开发者工具**（F12）查看 Console：

- `permission denied` → 用户不是管理员
- `relation "settings" does not exist` → settings 表未创建
- `Error loading site name` → 数据库连接失败

### 问题 3：保存设置时提示错误

**确认测试用户是管理员**：
```sql
-- 在 Supabase SQL Editor 执行
UPDATE profiles 
SET user_type = 'admin' 
WHERE email = 'test@example.com';
```
