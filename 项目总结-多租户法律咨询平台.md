## 📋 项目概述

### 项目目标
构建一个**多租户 SaaS 法律咨询平台**，支持：
1. 多个律所/法律服务机构独立运营
2. 每个租户拥有独立品牌（贴牌/White-Label）
3. 统一的 Super Admin 管理所有租户
4. 完整的用户咨询、律师管理、订单处理功能

### 系统架构

## 📊 数据库设计

### 核心表结构

#### 1. profiles（用户表）
```sql
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  email TEXT UNIQUE NOT NULL,
  full_name TEXT,
  user_type TEXT DEFAULT 'user',  -- 'user', 'admin', 'lawyer'
  super_admin BOOLEAN DEFAULT false,
  tenant_id UUID REFERENCES tenants(id),
  avatar_url TEXT,
  phone TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 2. tenants（租户表）
```sql
CREATE TABLE tenants (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  subdomain TEXT UNIQUE NOT NULL,
  custom_domain TEXT,
  status TEXT DEFAULT 'active',
  
  -- 贴牌配置
  branding JSONB DEFAULT '{
    "logo_url": "",
    "primary_color": "#FF6B35",
    "secondary_color": "#004E89",
    "company_name": "",
    "tagline": ""
  }',
  
  -- 功能配置
  features JSONB DEFAULT '{
    "consultations": true,
    "lawyers": true,
    "articles": true,
    "templates": true
  }',
  
  -- 联系信息
  contact_info JSONB DEFAULT '{
    "email": "",
    "phone": "",
    "address": ""
  }',
  
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 3. consultations（咨询表）
```sql
CREATE TABLE consultations (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  tenant_id UUID REFERENCES tenants(id) NOT NULL,
  user_id UUID REFERENCES profiles(id) NOT NULL,
  lawyer_id UUID REFERENCES profiles(id),
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  category TEXT NOT NULL,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 4. lawyers（律师表）
```sql
CREATE TABLE lawyers (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  tenant_id UUID REFERENCES tenants(id) NOT NULL,
  user_id UUID REFERENCES profiles(id) NOT NULL,
  specialization TEXT[],
  experience_years INTEGER,
  rating DECIMAL(3,2),
  bio TEXT,
  verified BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### RLS 策略

#### 简化版策略（避免循环依赖）
```sql
-- 用户可以查看自己的 profile
CREATE POLICY "simple_select_own" ON profiles
  FOR SELECT
  TO authenticated
  USING (auth.uid() = id);

-- 租户数据隔离
CREATE POLICY "tenant_isolation" ON consultations
  FOR SELECT
  TO authenticated
  USING (
    tenant_id = (
      SELECT tenant_id FROM profiles WHERE id = auth.uid()
    )
  );
```

## 🔐 认证系统

### 三层权限体系

#### 1. Super Admin（超级管理员）
- **权限**：管理所有租户、用户、系统配置
- **标识**：`super_admin = true`, `tenant_id = NULL`
- **登录入口**：`/super-admin/login`
- **Dashboard**：`/super-admin/dashboard-simple`

#### 2. Tenant Admin（租户管理员）
- **权限**：管理自己租户的用户、律师、咨询
- **标识**：`user_type = 'admin'`, `tenant_id = [租户ID]`
- **登录入口**：`/admin/login`
- **Dashboard**：`/admin`

#### 3. User（普通用户）
- **权限**：查看和创建咨询、查看律师
- **标识**：`user_type = 'user'`, `tenant_id = [租户ID]`
- **登录入口**：前台网站登录

### 认证流程

#### 登录流程（关键）
```typescript
// 1. 登录前先登出（避免 session 混淆）
await supabase.auth.signOut();

// 2. 登录新账号
const { data, error } = await supabase.auth.signInWithPassword({
  email,
  password,
});

// 3. 将 session 写入 HTTP-only Cookie
await fetch('/api/auth/callback', {
  method: 'POST',
  body: JSON.stringify({
    access_token: data.session.access_token,
    refresh_token: data.session.refresh_token,
  }),
});

// 4. 等待 Cookie 生效
await new Promise(resolve => setTimeout(resolve, 100));

// 5. 硬刷新跳转（确保 Server Component 读取最新 Cookie）
window.location.href = '/dashboard';
```

#### 权限检查（Server Component）
```typescript
// Layout 中进行权限检查
export default async function ProtectedLayout({ children }) {
  const supabase = await createServerClient();
  const { data: { session } } = await supabase.auth.getSession();
  
  if (!session) {
    redirect('/login');
  }
  
  const { data: profile } = await supabase
    .from('profiles')
    .select('*')
    .eq('id', session.user.id)
    .maybeSingle();
  
  if (!profile?.super_admin) {
    redirect('/login');
  }
  
  return <>{children}</>;
}
```

## 🐛 已解决的关键问题

### 1. 无限重定向循环 ✅
**问题**：访问 `/super-admin/login` 时出现 `ERR_TOO_MANY_REDIRECTS`

**原因**：Layout 覆盖了登录页路径

**解决方案**：使用 Route Groups
```
src/app/super-admin/
├── (auth)/                    ← 带权限检查的 Route Group
│   ├── layout.tsx             ← 权限检查 Layout
│   └── dashboard-simple/
│       └── page.tsx           ← 受保护页面
└── login/
    └── page.tsx               ← 独立登录页
```

### 2. RLS 策略循环依赖 ✅
**问题**：Session 存在但 Profile 无法读取

**原因**：RLS 策略中查询同一张表导致循环依赖

**解决方案**：简化 RLS 策略
```sql
-- ✅ 正确的策略
CREATE POLICY "simple_select_own" ON profiles
  FOR SELECT
  TO authenticated
  USING (auth.uid() = id);
```

### 3. Session 管理问题 ✅
**问题**：登录成功但无法访问受保护页面

**原因**：Client Component 和 Server Component 的 Session 不同步

**解决方案**：
1. 登录前先 signOut
2. 写入 HTTP-only Cookie
3. 等待 Cookie 生效
4. 使用 window.location.href 硬刷新跳转
