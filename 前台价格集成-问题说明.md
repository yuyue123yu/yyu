# 前台价格集成问题

## 🔍 问题描述

**现象**：
- 在 Admin 后台修改价格配置后
- 前台首页的价格没有更新
- 仍然显示硬编码的默认价格

## 🎯 根本原因

前台组件使用了硬编码的价格数据，没有从数据库读取租户配置。

### 问题文件

**`src/components/home/Services.tsx`**：
```typescript
const services = [
  {
    icon: Scale,
    titleKey: "services.debt",
    avgPrice: "RM 3,500",  // ❌ 硬编码
    // ...
  },
  // ...
];
```

## 💡 解决方案

需要修改前台组件，让它从租户配置 API 读取价格数据。

### 方案 1：完整集成（推荐）

创建一个 API 端点，让前台可以读取租户的价格配置：

1. **创建公开 API**：`/api/public/tenant-config`
   - 不需要登录即可访问
   - 根据域名或子域名识别租户
   - 返回租户的价格、品牌、内容等配置

2. **修改前台组件**：
   - 使用 `useEffect` 从 API 获取配置
   - 用配置数据替换硬编码的价格
   - 添加加载状态和错误处理

### 方案 2：简化方案（快速测试）

暂时使用默认租户的配置：

1. 修改前台组件直接调用现有的 `/api/tenant/pricing` API
2. 使用第一个租户的配置作为默认值

## 📋 需要修改的文件

### 前台组件
1. `src/components/home/Services.tsx` - 热门服务价格
2. `src/components/home/Hero.tsx` - 首页横幅的咨询价格（RM 99, RM 299, RM 999）
3. `src/app/services/page.tsx` - 服务列表页面的价格

### API 端点
1. 创建 `src/app/api/public/tenant-config/route.ts` - 公开的租户配置 API

## 🚀 实施步骤

### 步骤 1：创建公开 API

创建一个不需要登录的 API，返回租户配置：

```typescript
// src/app/api/public/tenant-config/route.ts
export async function GET(request: Request) {
  // 1. 从请求头获取域名
  // 2. 根据域名查找租户
  // 3. 返回租户的价格、品牌等配置
}
```

### 步骤 2：修改前台组件

修改 `Services.tsx` 组件：

```typescript
"use client";

import { useEffect, useState } from 'react';

export default function Services() {
  const [pricing, setPricing] = useState(null);
  
  useEffect(() => {
    // 从 API 获取租户配置
    fetch('/api/public/tenant-config')
      .then(res => res.json())
      .then(data => setPricing(data.pricing));
  }, []);
  
  // 使用 pricing 数据替换硬编码的价格
}
```

### 步骤 3：测试

## ⚠️ 注意事项

### 多租户支持

如果系统支持多租户（不同域名显示不同租户的配置）：

1. **子域名识别**：
   - `tenant1.example.com` → 租户 1 的配置
   - `tenant2.example.com` → 租户 2 的配置

2. **自定义域名识别**：
   - `www.lawfirm1.com` → 租户 1 的配置
   - `www.lawfirm2.com` → 租户 2 的配置

3. **默认租户**：
   - `localhost:3000` → 使用第一个租户的配置
   - `example.com` → 使用默认租户的配置

### 缓存策略

为了提高性能，可以添加缓存：

1. **浏览器缓存**：
   - 使用 `Cache-Control` 头
   - 缓存时间：5-10 分钟

2. **服务器缓存**：
   - 使用 Redis 或内存缓存
   - 当租户更新配置时清除缓存
