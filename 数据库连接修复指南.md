## 🐛 常见问题和解决方案

### 问题1：环境变量未加载

**症状：**
```
数据库连接失败
Error: Invalid Supabase URL
```

**解决方案：**
1. 确认 `.env.local` 文件存在
2. 重启开发服务器：
   ```bash
   # 停止服务器 (Ctrl + C)
   # 重新启动
   npm run dev
   ```

---

### 问题2：数据库表不存在

**症状：**
```
数据库查询失败
Error: relation "tenants" does not exist
```

**解决方案：**
1. 运行数据库迁移脚本（见步骤4）
2. 或手动创建表

---

### 问题3：权限不足

**症状：**
```
数据库查询失败
Error: permission denied for table tenants
```

**解决方案：**
1. 检查 RLS 策略是否正确
2. 确认使用了正确的 API Key
3. 运行 RLS 策略脚本：`supabase/007_create_rls_policies.sql`

---

### 问题4：网络连接问题

**症状：**
```
数据库连接失败
Error: fetch failed
```

**解决方案：**
1. 检查网络连接
2. 确认 Supabase 服务状态：https://status.supabase.com
3. 检查防火墙设置

---

### 问题5：API Key 过期或无效

**症状：**
```
数据库连接失败
Error: Invalid API key
```

**解决方案：**
1. 登录 Supabase Dashboard
2. 进入 Settings > API
3. 复制新的 API Keys
4. 更新 `.env.local` 文件
5. 重启开发服务器
