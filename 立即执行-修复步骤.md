# 🚀 立即执行 - 修复登录问题

## 📋 问题原因

您切换了Kiro账号，可能导致：
- Supabase项目连接不匹配
- 数据库修复应用到了错误的项目

## ✅ 修复步骤（按顺序执行）

### 步骤1：访问诊断页面

1. **启动开发服务器**（如果未启动）
   ```bash
   npm run dev
   ```

2. **打开诊断页面**
   ```
   http://localhost:3000/diagnose-login
   ```

3. **查看诊断结果**
   - 记录所有显示为 ❌ 的项目
   - 特别注意"环境变量"部分的 URL

---

### 步骤2：验证Supabase项目

1. **打开Supabase Dashboard**
   - 访问：https://supabase.com/dashboard
   - 登录您的账号

2. **找到正确的项目**
   - 查看项目列表
   - 找到 LegalMY 项目（或您的项目名称）

3. **复制项目信息**
   - 点击项目进入
   - 进入 Settings > API
   - 复制以下信息：
     - Project URL（例如：`https://xxxxx.supabase.co`）
     - anon public key
     - service_role key

4. **对比 `.env.local` 文件**
   - 打开项目根目录的 `.env.local` 文件
   - 对比 `NEXT_PUBLIC_SUPABASE_URL` 是否匹配
   - **如果不匹配，继续步骤3**
   - **如果匹配，跳到步骤4**

---

### 步骤3：更新环境变量（如果项目不匹配）

1. **更新 `.env.local` 文件**
   
   将以下内容替换为您从Supabase Dashboard复制的信息：
   
   ```env
   # Supabase Configuration
   NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=YOUR_ANON_KEY_HERE
   SUPABASE_SERVICE_ROLE_KEY=YOUR_SERVICE_ROLE_KEY_HERE

   # App Configuration
   NEXT_PUBLIC_APP_URL=http://localhost:3000
   ```

2. **清除缓存并重启服务器**
   
   在终端中执行：
   ```bash
   # 停止当前服务器（按 Ctrl+C）
   
   # 删除 .next 文件夹
   rm -rf .next
   
   # 重新启动
   npm run dev
   ```

3. **重新访问诊断页面**
   ```
   http://localhost:3000/diagnose-login
   ```
   
   确认"环境变量"部分显示 ✅

---

### 步骤4：在Supabase中执行修复脚本

1. **打开Supabase SQL Editor**
   - 在 Supabase Dashboard 中
   - 点击左侧菜单的 "SQL Editor"

2. **执行修复脚本**
   - 复制文件 `完整修复脚本-重新应用.sql` 的全部内容
   - 粘贴到 SQL Editor
   - 点击 "Run" 执行

3. **查看执行结果**
   - 应该看到多个 "Success" 消息
   - 最后会显示验证结果
   - 确认管理员账号存在

## 🚨 如果仍然失败

### 方案A：查看详细错误

1. **打开浏览器开发者工具**
   - 按 F12
   - 切换到 Console 标签

2. **尝试登录**
   - 记录所有红色错误信息

3. **切换到 Network 标签**
   - 尝试登录
   - 查看失败的请求
   - 点击失败的请求查看详情

### 方案B：创建新的管理员账号

1. **在Supabase Dashboard中**
   - Authentication > Users
   - 点击 "Add user"
   - 创建新用户（例如：`admin@test.com`）

2. **在SQL Editor中执行**
   ```sql
   -- 替换 USER_ID 为新用户的ID
   UPDATE profiles
   SET user_type = 'admin', super_admin = true, tenant_id = NULL
   WHERE id = 'USER_ID_HERE';
   ```

3. **使用新账号登录**
