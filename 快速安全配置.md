## 🔒 第二步：在登录页面使用限流（10分钟）

### 修改 Admin 登录

找到 `src/app/admin/login/page.tsx`，添加限流检查：

```typescript
import { checkLoginAttempts, resetLoginAttempts } from '@/lib/rateLimit';

const handleLogin = async (e: FormEvent) => {
  e.preventDefault();
  
  // 1. 检查登录尝试次数
  const rateCheck = checkLoginAttempts(email);
  
  if (!rateCheck.allowed) {
    const blockedMinutes = Math.ceil(
      (rateCheck.blockedUntil!.getTime() - Date.now()) / 60000
    );
    setError(`登录尝试次数过多，请${blockedMinutes}分钟后再试`);
    return;
  }
  
  // 2. 尝试登录
  const { error } = await supabase.auth.signInWithPassword({
    email,
    password,
  });
  
  if (error) {
    setError('用户名或密码错误');
    // 显示剩余尝试次数
    if (rateCheck.remainingAttempts !== undefined) {
      setError(`用户名或密码错误。剩余尝试次数：${rateCheck.remainingAttempts}`);
    }
    return;
  }
  
  // 3. 登录成功，重置尝试次数
  resetLoginAttempts(email);
  router.push('/admin');
};
```

## 🛡️ 第三步：添加输入验证（10分钟）

### 在表单中使用验证

示例：用户注册表单

```typescript
import { validateEmail, validatePassword, sanitizeInput } from '@/lib/validation';

const handleRegister = async (e: FormEvent) => {
  e.preventDefault();
  
  // 1. 验证邮箱
  if (!validateEmail(email)) {
    setError('邮箱格式不正确');
    return;
  }
  
  // 2. 验证密码强度
  const passwordCheck = validatePassword(password);
  if (!passwordCheck.valid) {
    setError(passwordCheck.errors.join(', '));
    return;
  }
  
  // 3. 清理输入（防止XSS）
  const cleanName = sanitizeInput(name);
  const cleanCompany = sanitizeInput(company);
  
  // 4. 提交注册
  const { error } = await supabase.auth.signUp({
    email,
    password,
    options: {
      data: {
        name: cleanName,
        company: cleanCompany,
      },
    },
  });
  
  if (error) {
    setError(error.message);
    return;
  }
  
  // 注册成功
  router.push('/verify-email');
};
```

## ⚙️ 第四步：配置 Supabase 安全（5分钟）

### 1. 启用邮箱验证

在 Supabase Dashboard：
1. Authentication → Settings
2. 找到 "Email Auth"
3. ✅ 勾选 "Enable email confirmations"
4. 保存

### 2. 配置密码策略

在同一页面：
1. 找到 "Password"
2. 设置：
   - Minimum password length: **8**
   - ✅ Require uppercase letters
   - ✅ Require lowercase letters
   - ✅ Require numbers
   - ✅ Require special characters
3. 保存

### 3. 检查 RLS 策略

在 SQL Editor 执行：
```sql
-- 检查所有表是否启用了 RLS
SELECT 
  schemaname, 
  tablename, 
  rowsecurity as rls_enabled
FROM pg_tables 
WHERE schemaname = 'public'
ORDER BY tablename;
```

确保所有表的 `rls_enabled` 都是 `true`。
