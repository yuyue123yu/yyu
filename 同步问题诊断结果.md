## 🔧 技术细节

### 为什么会出现这个问题？

1. **迁移文件未执行**
   - `create-settings-table.sql` 存在但没有编号
   - 不在标准迁移序列中（001-010）
   - 可能被遗漏了

2. **Admin 保存失败但未提示**
   - Admin 尝试保存到不存在的表
   - 错误被 catch 但只在控制台显示
   - 用户看到"保存成功"但实际失败

3. **前端读取失败**
   - SiteSettingsContext 尝试读取不存在的表
   - 降级到默认值 "LegalMY"
   - 所以前端始终显示默认值

### settings 表结构

```sql
CREATE TABLE public.settings (
  id UUID PRIMARY KEY,
  key TEXT UNIQUE NOT NULL,      -- 设置键（如 'site'）
  value JSONB NOT NULL,           -- 设置值（JSON格式）
  updated_at TIMESTAMP,
  updated_by UUID
);
```

### 默认数据

```json
{
  "key": "site",
  "value": {
    "siteName": "马来西亚法律网",
    "siteDescription": "专业法律咨询平台",
    "contactEmail": "support@legalmy.com",
    "contactPhone": "+60 3-1234 5678",
    "defaultLanguage": "zh"
  }
}
```
